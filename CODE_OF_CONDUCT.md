# Code of Conduct
## Our Mission
The goal of the **bf16** project is to provide highly optimized code for Windows x64 Python developers who need to use the BFloat16 format. We aim to be **Safe, Simple, and Efficient**, moving data as fast as possible while keeping the implementation transparent and bloat-free.

## Our Standards
To maintain a high-quality technical environment, we expect all participants to:
*   **Be Kind:** Treat every contributor and user with basic human decency.
*   **Be Clear:** Make sure people can understand you. Use clear language and provide enough context so others can help or learn.
*   **Be Respectful:** Value the time and effort put into this project. Constructive technical debate is encouraged; personal attacks are not.

## Enforcement
As the project owner, I will take the following actions to ensure the project remains a productive and optimized space:
1.  **Comment Moderation:** I will edit or delete comments that are rude, unhelpful, or low-effort.
2.  **Issue Management:** I will close Issues that do not provide constructive feedback or technical value.
3.  **Account Access:** I will block users who repeatedly violate these rules or intentionally create a toxic environment.
